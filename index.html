1. What is creation phase and execution phase?

creation phase:

Compiler runs through the entire code for 2 time before actually executing the code,

  • In the first run, It picks all function declarations and stores them in memory with their reference.
  • In the second run, It picks all variables and assign undefined to them. In the event of a conflict between variable and function declaration name then that variable is ignored.

Execution phase:

Finally, right after the creation phase of an Execution Context comes the execution phase. This is the stage where the actual code execution begins, Variables assigned with values and Functions executed.

2. What are objects in javascript?

In JS, Object is a non-primitive datatypethat allows you to store multiple collection of data in the form of key-value pairs.

1. object literal

        let obj = {
            name: "Dhoni"
            age: 43,
        }
        console.log(obj.name);

2. creating instance:

        let obj1 = new Object();
        obj1.name = "Bhanu";
        obj1.age = 21,

3. object constructor

        function Obj(name, age){
            this.name = name;
            this.age = age;
        }
        let obj1 = new Obj('pavan' , 23)
        console.log(obj1)

4. Explain prototypes?

Prototype is itself an object that have prototype object present inside it.

        function Student(name, age, rollno){
            this.name = name;
            this.age = age;
            this.rollno = rollno;
        };
        let student1 = new Student('Bhanu', 21, 115);
        let student2 = new Student('pavan', 23, 118);
        console.log(student1);

5. What is prototype chain?

All objects in JavaScript are instances of Object. Objects created using new keyword, inherit from a prototype called Object.prototype. That means all the objects in JavaScript, inherit the properties and methods from Object.prototype. This is called Prototype chaining.

7. What are callbacks?

Function that is passed as an argument to another function is known as allback function. As JS is event driven language which means they don't stop the program from running until they are completed.
